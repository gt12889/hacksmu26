# v1.0 Milestone Audit — ElephantVoices Denoiser

## Summary

**Status:** gaps_found
**Requirements satisfied:** 38/48 (79%)

**Phases complete:** 6/6

All 6 phases completed execution. The core DSP pipeline is solid and all 171 tests pass. However, the Phase 5 React frontend merge (adopting teammate's remote-branch implementation) dropped the upload/result UI path that our local Phase 5 plan had built. The live demo currently works end-to-end only for the pre-generated 3-noise-type static demo, not for user-uploaded recordings.

## What Works (Shippable)

- ✅ **CLI pipeline** — ingest → process → demo_spectrograms → batch_process end-to-end
- ✅ **Publication figures** — 3 noise types at 300dpi with f0 contour, harmonic markers, SNR annotations
- ✅ **Multi-speaker separation** — validated on synthetic 14+18Hz mixture, produces 2 colored tracks + separate WAVs
- ✅ **FastAPI backend** — 15 routes running, 14 API tests passing
- ✅ **React demo UI** — teammate's ElephantVoices-branded frontend displays the 3 noise types
- ✅ **Scientific moat** — HPSS + SHS + comb mask + octave-check + NSSH all implemented correctly
- ✅ **171 tests passing** — no regressions across phases

## Critical Gaps

### 1. Upload flow unreachable (UI-01..05)
Frontend `App.tsx` (from merged teammate branch) implements only the static 3-noise-type demo. The planned upload/process/result path with UploadPanel/CallDetail/ConfidenceTable/SpectrogramView is fully built in component files but never imported.
**Fix:** Either integrate those components into App.tsx, or explicitly descope UI-01..05 to "static demo only" and retest.

### 2. `_original.wav` missing (DEMO-06, UI-04)
Frontend expects `/static/demo/{type}_original.wav` for the A/B audio toggle. `scripts/demo_spectrograms.py` only writes `_clean.wav` and `_demo.png`.
**Fix:** Add `sf.write(f"{type}_original.wav", y_noisy, sr)` in `make_demo_figure()`.

### 3. Vite proxy port split
`vite.config.ts` proxies `/api` to port 8001 and `/static` to port 8000. FastAPI is single-port.
**Fix:** Set both proxy targets to same port, or add a clear run script.
